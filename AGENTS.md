# Repository instructions

This repository builds a premium personal website for Dhruv Mehta.

## Priority order
1. Truthfulness to the source resume
2. Exceptional UI/UX quality
3. Responsive polish
4. Accessibility and performance
5. Clean deployment on GitHub Pages

## Identity / positioning
Lead with Dhruv's present-day identity:
- business continuity
- information security
- business analysis

Use machine learning as technical depth and educational foundation, not as the primary headline.

## Design rules
- Avoid generic, overbuilt layouts.
- No hero cards.
- No dashboard-looking hero.
- The first viewport must read as one composition.
- One strong visual anchor in the hero.
- No fake social proof, no fake metrics, no invented claims.
- Use at most two typefaces in the shipped site.
- Use gradients and subtle atmospheric layers intentionally.
- Prefer elegant SVG / CSS visuals over stock imagery.
- Keep spacing generous and hierarchy clear.

## Content rules
Use only facts from the provided resume content.
Do not invent certifications, achievements, leadership scope, or extra projects.

## Build rules
- Use React + TypeScript + Vite + Tailwind.
- Create a design token system early.
- Keep code modular and readable.
- Use semantic HTML.
- Ensure mobile quality is first-class.
- Add reduced-motion support.

## Verification
Before finishing:
- run typecheck
- run build
- run lint if configured
- verify the site on desktop and mobile breakpoints
- verify all links and buttons
- verify GitHub Pages deployment config
